Este diseño impide que el software realice cálculos automáticos, búsquedas por producto o filtros de stock, ya que los datos están mezclados.

3. Institucionalización: La Primera Forma Normal (1FN)

Para que una base de datos sea profesional, debe cumplir con la regla de Atomicidad.

Dominio Atómico: Cada celda de una tabla debe contener un único valor indivisible para el sistema. Se prohíben los "grupos repetitivos" (listas de valores dentro de una sola celda).
Primera Forma Normal (1FN): Una tabla está en 1FN si y solo si todos sus atributos son atómicos. Esto permite transformar el caos de la información primaria en una estructura lógica organizada.

4. ¿Para qué sirve y qué riesgos corremos?

¿Para qué sirve?
Permite que el sistema realice cálculos precisos, genere facturas automáticas y facilite la búsqueda de información individualizada.

Riesgos de no aplicar la 1FN:
  • Imposibilidad técnica: El software no puede operar con los datos de forma individual.
  • Redundancia y desorden: Se genera un "caos informativo" que dificulta el mantenimiento.
  • Inconsistencia: Es extremadamente difícil actualizar un solo producto si está mezclado con otros.

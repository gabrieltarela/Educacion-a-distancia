Proyecto Final: Diseño de una Biblioteca Profesional

El Escenario

Has sido contratado para organizar el sistema de información de una pequeña biblioteca. Actualmente, el bibliotecario registra todo de forma desordenada en una sola lista de Excel. Tu misión es aplicar el algoritmo de refinamiento que hemos estudiado para transformar este caos en una base de datos eficiente, profesional y sin errores.

1. La Información Primaria (El Caos)

Este es el listado de atributos que el bibliotecario tiene mezclados en una sola estructura:

(Libro, Autor, Nacionalidad_Autor, Editorial, Socio, Fecha_Préstamo)
¡Atención! Este diseño actual genera anomalías: por ejemplo, si borras a un socio, ¡podrías perder la información del libro que tenía prestado! Tu objetivo es evitar esto mediante la normalización.

2. Tu Desafío: Consigna de Trabajo

En subgrupos, deben resolver el problema siguiendo estos pasos técnicos:

1
Identidad Única: Define las Claves Primarias (PK) necesarias para cada entidad, garantizando el criterio de minimalidad.
2
Atomicidad (1FN): Asegúrate de que cada celda contenga un valor único e indivisible. No se permiten listas de autores o libros en una misma celda.
3
Especialización (2FN): Elimina cualquier dependencia parcial. Si un dato depende solo de una "parte" de una clave compuesta, sepáralo en una nueva tabla.
4
Determinación Directa (3FN): Elimina el "efecto dominó" o dependencias transitivas. Cada dato debe estar exclusivamente donde su determinante directo se encuentra.

3. Producto Final

Deben entregar un esquema relacional optimizado (diagrama de tablas y flechas) que represente visualmente la estructura normalizada de la biblioteca.

Preguntas de Metacognición

Al finalizar su diseño, respondan grupalmente:

  • ¿En qué momento del proceso se dieron cuenta de que una tabla necesitaba ser "partida" para ser más eficiente?.
  • ¿Qué anomalías de datos (errores al borrar o actualizar) lograron evitar con su diseño final?.
